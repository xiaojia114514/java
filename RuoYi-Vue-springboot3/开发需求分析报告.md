# 课程目标达成情况分析报告系统开发需求分析

## 一、需求背景

根据提供的文档（教学大纲、成绩单模板、试卷实质审查表、课程教学质量分析报告模板），开发一个辅助教师生成"课程目标达成情况分析报告"的系统，实现教学目标与试卷关联、学生成绩自动导入、达成度计算、报告自动生成等功能，利用LLM生成并优化报告表述。

## 二、系统功能需求

### 1. 核心功能模块

#### 1.1 教学大纲管理
- 导入/存储教学大纲（Word文档）
- 提取并管理课程目标
- 支持课程目标的增删改查
- 教学大纲版本管理
- 仅存储文档元数据，不修改原始文档内容

#### 1.2 试卷管理
- 导入/维护试卷信息
- 试卷与教学目标关联
- 试题与教学目标关联
- 试卷实质审查表管理

#### 1.3 成绩管理
- 自动导入学生成绩单（支持Excel格式）
- 成绩数据验证与清洗
- 成绩统计分析

#### 1.4 达成度计算
- 基于教学目标与试题关联计算达成度
- 支持多种达成度计算方法
- 生成达成度分析图表

#### 1.5 报告生成
- 自动生成课程目标达成情况分析报告
- 利用LLM优化报告表述
- 支持报告模板定制
- 报告导出（Word、PDF格式）

### 2. 辅助功能

#### 2.1 用户管理
- 基于若依框架的用户管理
- 角色权限控制

#### 2.2 数据可视化
- 达成度分析图表
- 成绩分布图表
- 教学目标达成趋势

#### 2.3 系统设置
- 达成度计算规则配置
- 报告模板配置
- LLM API配置

### 2. 关联关系

- **课程 - 目标**：一对多 (course_id)
- **课程 - 试卷**：一对多 (course_id)
- **试卷 - 试题**：一对多 (paper_id)
- **试题 - 目标**：多对多 (通过question_goal表)
- **试卷 - 学生成绩**：一对多 (paper_id)
- **学生 - 成绩**：一对多 (user_id)
- **目标 - 达成度**：一对多 (goal_id)
- **课程 - 报告**：一对多 (course_id)

## 五、系统架构设计

### 1. 技术栈

#### 1.1 前端
- Vue 3
- Element Plus
- ECharts (数据可视化)
- Axios (HTTP请求)
- Vue Router (路由)

#### 1.2 后端
- Spring Boot 3
- MyBatis-Plus
- MySQL
- Spring Security (权限控制)
- Apache POI (Excel处理)
- OpenAI API (LLM集成)

#### 1.3 部署
- 服务器：Tomcat
- 数据库：MySQL
- 存储：本地文件系统或OSS

### 2. 系统流程

#### 2.1 数据准备流程
1. 导入教学大纲，提取课程目标
2. 创建试卷，关联试题与课程目标
3. 导入学生成绩单
4. 系统自动计算达成度

#### 2.2 报告生成流程
1. 选择课程和试卷
2. 系统计算达成度统计数据
3. LLM生成报告内容
4. 教师编辑报告
5. 导出报告文件

## 九、风险评估

### 1. 风险因素
- 数据导入格式不一致
- 达成度计算算法复杂性
- LLM API调用稳定性
- 系统性能问题

### 2. 应对策略
- 提供多种数据导入格式支持
- 设计灵活的达成度计算配置
- 实现API调用失败的容错机制
- 优化数据库查询和计算性能

## 十、结论

本系统将有效解决教师在课程目标达成情况分析中的痛点，通过自动化的数据处理、智能的达成度计算和报告生成，大大减轻教师的工作负担，提高教学评估的准确性和效率。系统设计充分考虑了扩展性和可维护性，为未来功能迭代和技术升级预留了空间。
