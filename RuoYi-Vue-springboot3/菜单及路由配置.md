## 四、权限控制配置

### 1. 权限标识设计
- **系统管理**：`system:user:list`, `system:role:list`, `system:menu:list`, `system:dept:list`, `system:config:list`
- **课程管理**：`system:course:list`, `system:syllabus:list`, `system:goal:list`, `system:paper:list`
- **成绩管理**：`system:score:import`, `system:score:list`, `system:achievement:calculate`
- **分析报告**：`system:report:generate`, `system:report:list`

### 2. 角色权限分配
| 权限标识 | 管理员 | 教师 |
|---------|--------|------|
| system:user:list | ✅ | ❌ |
| system:role:list | ✅ | ❌ |
| system:menu:list | ✅ | ❌ |
| system:dept:list | ✅ | ❌ |
| system:config:list | ✅ | ❌ |
| system:course:list | ✅ | ✅ |
| system:syllabus:list | ✅ | ✅ |
| system:goal:list | ✅ | ✅ |
| system:paper:list | ✅ | ✅ |
| system:score:import | ✅ | ✅ |
| system:score:list | ✅ | ✅ |
| system:achievement:calculate | ✅ | ✅ |
| system:report:generate | ✅ | ✅ |
| system:report:list | ✅ | ✅ |

### 3. 前端权限控制
- 使用 Vue Router 的 meta 属性配置权限标识
- 使用自定义指令 `v-hasPermi` 控制按钮和菜单的显示
- 在路由守卫中进行权限验证

## 五、配置建议

1. **菜单图标**：使用 Element Plus 提供的图标，如 `education`、`file-text`、`goal` 等
2. **路由地址**：使用语义化的路径，如 `/course/list`、`/score/import` 等
3. **组件路径**：按照模块组织，如 `@/views/course/index.vue`、`@/views/score/import.vue` 等
4. **权限标识**：采用 `模块:功能:操作` 的格式，如 `system:course:list`
5. **缓存设置**：对于频繁访问的页面开启缓存，如课程列表、成绩管理等
6. **显示排序**：根据功能重要性设置合理的显示顺序

## 六、实施步骤

1. **后端配置**：
   - 在 `sys_menu` 表中添加新的菜单和权限
   - 在 `sys_role_menu` 表中为不同角色分配权限

2. **前端配置**：
   - 配置路由文件，添加新的路由
   - 创建对应的组件页面
   - 实现权限控制逻辑

3. **测试验证**：
   - 使用不同角色登录系统
   - 验证菜单显示是否正确
   - 验证权限控制是否有效

此菜单和路由配置设计符合系统功能需求，同时考虑了不同角色的权限差异，确保系统的安全性和易用性。
